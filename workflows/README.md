# Workflows

Markdown SOPs that define what to do and how. Each workflow should specify:

- **Objective** — what this workflow accomplishes
- **Required inputs** — what's needed before starting
- **Tools** — which scripts in `tools/` to run, and in what order
- **Expected outputs** — what done looks like, and where deliverables go
- **Edge cases** — known failure modes and how to handle them

Name files descriptively, e.g. `scrape_website.md`, `generate_report.md`.
